# 阶段2-WP2：搜索 UI 与快捷添加

## 目标

在 `/add` 页面上叠加 Bangumi 搜索能力，实现搜索 → 选状态 → 快捷添加的完整流程。

## 前置依赖

* WP1（API 客户端）必须先完成

## 范围

* `/add` 页面顶部搜索栏
* 搜索 debounce + 状态管理（Riverpod）
* 搜索结果卡片组件（封面、标题、年份、媒介类型、简介）
* 已在库指示器（通过 `sourceIdsJson` 匹配）
* 状态选择层 dialog
* `BangumiQuickAddController`（从 Bangumi Subject 创建本地条目）
* "手动创建"入口保留
* 封面图远程加载（`cached_network_image`）
* 搜索空状态和错误状态

## 不包含

* Bangumi 账号绑定（WP3）
* 自动同步推送（WP3）

## 页面结构

```
┌────────────────────────────────────────────────────┐
│ 搜索栏 [输入关键词...] [类型筛选 ▼]               │
├────────────────────────────────────────────────────┤
│ 搜索结果                                           │
│ ┌──────┬───────────────────────────────────────┐   │
│ │ 封面 │ 标题 / 副标题 / 年份 / 类型标签       │   │
│ │ 缩略 │ 简介摘要...                     [添加] │   │
│ └──────┴───────────────────────────────────────┘   │
│ ┌──────┬───────────────────────────────────────┐   │
│ │ 封面 │ 标题 / 副标题 / 年份 / 类型标签       │   │
│ │ 缩略 │ 简介摘要...                  [已添加] │   │
│ └──────┴───────────────────────────────────────┘   │
├────────────────────────────────────────────────────┤
│ ─── 或手动创建 ───                                │
│ [展开 WP4 已有的手动创建表单]                      │
└────────────────────────────────────────────────────┘
```

## 快捷添加流程

1. 用户点击搜索结果的"添加"
2. 弹出状态选择 dialog（5 个 UnifiedStatus 选项）
3. 选择后 controller 完成：
   * 映射 Bangumi Subject → MediaItem（填充所有元数据 + `sourceIdsJson` + `posterUrl`）
   * 创建默认 UserEntry（写入选择的状态）
   * 追加 ActivityLog
4. 轻反馈"已加入本地"
5. 搜索结果更新为"已添加"状态
6. 用户可继续搜索

## 验收标准

* [ ] 搜索栏输入后 debounce 调用 Bangumi API
* [ ] 搜索结果展示封面、标题、副标题、年份、媒介类型、简介
* [ ] 支持按媒介类型筛选（默认全部，筛选器对应 Bangumi `type: [1, 2, 4, 6]`，排除 music=3）
* [ ] 点击添加弹出状态选择层
* [ ] 选择状态后写入本地库，`sourceIdsJson` 含 Bangumi ID
* [ ] 已在库条目显示当前状态标签
* [ ] 添加成功后的轻反馈里提供"查看详情"入口，点击可跳转到新条目的 `DetailPage`
* [ ] "手动创建"入口仍可使用
* [ ] 封面图正常加载，失败时 fallback 到占位色
* [ ] 搜索无结果有空状态
* [ ] 网络错误有错误状态

## 同步 hook 约定

WP2 的 `BangumiQuickAddController` 必须在本地写入成功后，调用由 WP3 注入的 `BangumiSyncService.pushCollection(...)`（通过构造函数或 provider 注入，WP2 本身不直接引用 token / API 客户端）。未绑定时，WP3 的默认实现静默跳过，WP2 不需要分支判断。

WP2 只负责调用，**不负责**：Token 读取、认证状态判断、失败重试、轻提示"已同步到 Bangumi"的发出——这些都由 WP3 的同步服务内部完成。
