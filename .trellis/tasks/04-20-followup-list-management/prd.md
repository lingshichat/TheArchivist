# 后续增强：自定义列表管理中心与批量管理

## 目标

在 WP4 已经打通“列表最小创建、挂接、回显”的前提下，把自定义列表从“能用”收成“好用”。

这个任务不再负责阶段1的本地闭环兜底，而是承接后续增强：

* 独立列表页
* 列表详情页
* 排序与整理
* 批量挂接 / 移除
* 更完整的列表管理体验

## 来源

本任务从 `04-19-phase1-wp4-local-record-flow` 拆出。

拆分原则：

* **留在 WP4 的**：最小列表链路
  * 创建列表
  * 给条目挂接 / 取消挂接
  * 在详情页回显
* **拆到本任务的**：高级列表管理
  * 独立列表页
  * 排序
  * 批量管理
  * 更完整的列表编辑体验

## 范围

* 独立“列表中心”入口与页面
* 列表卡片 / 列表墙 / 列表目录
* 列表详情页，展示该列表下的条目
* 列表重命名、删除、基础元信息编辑
* 列表内条目排序
* 批量加入列表 / 移出列表
* 空列表空状态
* 列表维度的筛选、计数和轻量统计

## 不包含

* WP4 的最小列表挂接链路
* Bangumi 同步
* 跨设备同步
* 协作列表 / 分享列表
* 智能推荐列表
* 列表封面拼图、复杂拖拽动画

## 验收标准

* [ ] 用户可以进入独立列表页浏览已有自定义列表
* [ ] 用户可以创建、重命名、删除自定义列表
* [ ] 用户可以在列表详情页看到该列表下的条目
* [ ] 用户可以批量把多个条目加入或移出某个列表
* [ ] 列表内支持明确的排序方式
* [ ] 空列表状态清楚，不退回默认 Material 管理台风格
* [ ] 与现有 Home / Library / Detail 的视觉契约保持一致

## 依赖

* 依赖 `04-19-phase1-wp4-local-record-flow`
* 依赖 WP4 已把 `ShelfRepository` 的最小链路接通

## 备注

这是后续增强任务，不阻塞 WP4 收口，也不阻塞阶段1最小本地闭环验收。
