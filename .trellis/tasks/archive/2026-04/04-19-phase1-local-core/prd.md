# 阶段1：本地可用的核心骨架

## 目标

把应用做成一个不开同步、不绑 Bangumi 也能独立使用的本地版本。

本阶段开发顺序采用：

* **Windows 优先**

原因：

* 当前项目首轮设计与调试在桌面端更高效
* 海报墙、详情页、双栏布局在 Windows 上更容易先定骨架
* 首轮先把信息架构和本地记录流打稳，再回到 Android 做移动端收口

本阶段平台策略：

* Windows：作为首版主交付端
* Android：本阶段只保留兼容性约束，不作为主要验收端

## 范围

* Flutter 工程初始化
* 主题系统与基础设计令牌
* Windows 优先的导航壳与页面骨架
* 本地数据模型与 `drift` 表设计
* 首页、库页、详情页基础结构
* 本地状态、评分、私人笔记、进度、列表、标签

## 不包含

* Bangumi 搜索与绑定
* WebDAV / S3-compatible 同步
* 自动同步与冲突处理
* Android 端细致适配与触控优化

## 视觉设计规范

### 视觉设计总纲

#### 设计目的

这个阶段不是做“功能页面集合”，而是先把这个产品的第一眼气质定住。

要解决的问题：

* 让个人记录工具看起来不枯燥
* 让海报墙成为真正的视觉锚点
* 让桌面端的信息密度高，但不显乱

#### 视觉论点

本阶段视觉论点定为：

* **像一个安静、克制、带收藏感的个人媒体陈列柜**

关键词：

* 克制
* 编辑感
* 收藏感
* 内容优先
* 桌面工作台气质

#### 风格极性

本阶段风格不走花哨，不走炫技，定为：

* **Refined / Editorial / Calm**

也就是：

* 不是玩具感
* 不是赛博炫彩
* 不是传统企业后台
* 不是完整原生 Fluent 复刻

### 视觉基线

本阶段采用：

* **Material 3 结构基线**
* **Windows 优先的桌面布局**
* **局部玻璃质感，不引入完整 Fluent 或完整 iOS 风格**

这样收的原因：

* 后续 Android 复用成本更低
* 不会因为过度平台化导致双端设计分裂
* 先把内容区、导航区、详情区层级打稳

#### 产品界面类型判断

这是产品工作面，不是营销页。

因此默认遵守：

* 先做主工作区，再做导航，再做次级信息区
* 少用装饰卡片，卡片只在它本身就是交互单元时出现
* 让海报本身承担品牌与氛围感，不额外堆太多装饰元素

### 本阶段设计目标

本阶段只解决三个问题：

1. 海报墙看起来顺
2. 详情页信息层级清楚
3. 记录动作入口清楚

#### 本阶段记忆点

用户看完首版界面后，应该记住一件事：

* **这不是表格型记录工具，而是一个可以浏览和沉淀个人媒体收藏的桌面空间**

### 本阶段视觉边界

允许优先设计的区域：

* 左侧导航
* 首页海报墙
* 详情页头图与信息区
* 状态、评分、进度操作区

允许使用轻玻璃的区域：

* 窗口顶部栏
* 搜索栏容器
* 浮层和弹窗

本阶段不要做的视觉动作：

* 整页毛玻璃背景
* 为了“桌面感”引入复杂阴影堆叠
* 先做 Android 风格再硬改成 Windows 风格
* 首页做成 KPI 仪表盘
* 把所有区域都包成厚重卡片
* 使用明显 AI 味很重的紫色渐变 SaaS 风

### 版式与构图规范

#### 首屏构图原则

首页首屏不是欢迎页，也不是营销 Hero。

首屏默认只做一件事：

* 让用户马上进入“继续看 / 最近加 / 最近完成”的内容工作面

首屏构图采用：

* 左侧稳定导航
* 右侧内容工作区
* 顶部保留轻量搜索与筛选入口
* 中心区域直接展示海报墙和内容分组

#### 空间层级

桌面端默认三层：

1. 导航层
2. 主内容层
3. 次级信息或操作层

层级要求：

* 导航层稳定，不抢内容
* 主内容层承担主要视觉重量
* 次级信息层只在详情页和编辑动作里增强，不常驻抢焦点

### Windows 首版布局规范

首版默认采用：

* 左侧 `NavigationRail`
* 右侧主内容区
* 首页与库页使用可扩展网格
* 详情页使用双栏布局

建议信息结构：

* 左栏：海报、状态、评分、进度、快捷操作
* 右栏：标题、简介、标签、笔记、历史

#### 页面宽度策略

Windows 首版按中到宽窗口优先设计。

默认规则：

* 容器不追求全宽铺满
* 内容区保留清楚的左右留白
* 超宽窗口下优先增加列数，不把单卡拉得过宽

### 组件外观规范

#### 海报卡

海报卡是核心交互单元。

规范：

* 优先展示封面，不先展示文字块
* 标题信息控制在 1 到 2 行
* 状态标签弱化，不压过封面
* Hover 时只做轻抬升、描边或阴影变化
* 不叠加复杂按钮矩阵

#### 详情操作区

详情操作区要做到：

* 状态操作一眼可见
* 评分和进度比笔记更靠前
* 私人笔记区不与主信息抢层级

#### 容器策略

默认尽量少卡片化。

只在这些场景使用明确卡片：

* 海报卡
* 浮层
* 设置中的同步配置块

普通信息分组更适合用：

* 留白
* 分隔
* 标题层级
* 对齐关系

### 设计令牌建议

首版先定以下基础令牌：

* 8pt 间距体系
* 三档圆角：卡片 / 容器 / 浮层
* 一套浅色主题先行
* 深色主题只保留结构预留，不作为本阶段重点

#### 字体规范

本阶段不直接用默认系统感很强的通用搭配作为品牌表达。

建议字体策略：

* 标题与数字：一套更有识别度的西文字体
* 正文与中文：一套清楚、稳的中文无衬线

建议方向：

* Heading / Number：`Space Grotesk` 或同级别几何无衬线
* Body / Chinese：`Noto Sans SC` 或同级别中文无衬线

要求：

* 控制在两套字以内
* 不为了风格牺牲中文可读性

#### 色彩规范

首版采用浅色主题先行，配一组克制的深色中性色。

颜色策略：

* 背景偏暖灰或中性灰白
* 内容区靠对比和层级，不靠高饱和背景
* 强调色只保留一组，用于状态、主操作、选中态

建议方向：

* Base：off-white / graphite / soft gray
* Accent：低饱和蓝绿或墨绿
* Warning / Error：只作为状态色，不扩散成装饰色

### 动效规范

本阶段至少保留 3 类有用动效：

1. 首屏内容渐入
2. 海报 hover 反馈
3. 详情区域切换或浮层展开

要求：

* 动效为层级服务，不做表演
* 时长短，节奏稳
* 不引入大面积模糊动画

建议方向：

* 海报 hover：轻抬升 + 阴影变化 + 边框增强
* 页面切换：淡入或轻滑入
* 浮层：纵向展开 + 透明度渐入

### 响应式收口规则

Windows 优先不等于只做固定宽度桌面页面。

本阶段至少保留三档响应式思路：

* 宽屏：详情页双栏、海报墙多列
* 中屏：详情页双栏但收窄间距
* 窄屏：详情页转上下结构，导航后续可切换为紧凑模式

## 交互观感规范

### 交互论点

整个产品的交互感受应当是：

* **轻、稳、少打断**

这意味着：

* 常用动作尽量贴近内容本身
* 反馈尽量轻提示
* 不做弹窗滥用

### 首版交互优先级

本阶段优先优化：

1. 海报点击进入详情
2. 详情内修改状态 / 评分 / 进度
3. 列表与详情之间往返

本阶段不优先优化：

* 高级快捷键系统
* 复杂拖拽
* 批量编辑

## 实施顺序

本阶段建议按下面顺序推进：

1. 工程初始化与主题骨架
2. Windows 导航壳与页面骨架
3. 本地数据表与仓储层
4. 首页、库页、详情页绑定本地数据
5. 状态、评分、进度、笔记写入链路
6. 基础空状态与错误状态
